Guia Completo: Tags Semânticas em HTML5

O que são Diferenças Vantagens Principais Tags Exemplo SEO & Acessibilidade Boas Práticas

1. O que são Tags Semânticas?

Tags semânticas são elementos HTML que carregam significado. Elas indicam qual o papel daquele conteúdo na página, facilitando a leitura para navegadores, leitores de tela e mecanismos de busca.

2. Diferença entre Tags Semânticas e Não-Semânticas

Semânticas: como <article>, <section> ou <footer>, têm significado claro.

Não-semânticas: como <div> e <span>, não dizem nada sobre o conteúdo.

3. Vantagens do Uso de Tags Semânticas

  • Melhor organização do código
  • Mais acessibilidade
  • Melhoria no SEO
  • Facilidade de manutenção

4. Principais Tags Semânticas do HTML5

<header>

Representa o cabeçalho da página ou de uma seção.

<nav>

Define a área de navegação (menu principal).

<main>

Contém o conteúdo principal da página. Só pode haver uma por página.

<section>

Usada para dividir a página em seções temáticas.

<article>

Representa um conteúdo independente, como uma postagem ou artigo.

<aside>

Conteúdo complementar, como uma barra lateral.

<footer>

Rodapé da página ou de uma seção.

<figure> e <figcaption>

Usados para imagens e legendas.

Outras Tags:

  • <mark>: destaca o texto
  • <time>: representa tempo ou data
  • <address>: informações de contato

6. Como as Tags Semânticas Ajudam no SEO e Acessibilidade

SEO: Ajuda mecanismos de busca a entenderem o conteúdo da página.

Acessibilidade: Facilita o uso por pessoas com deficiência, através de leitores de tela.

7. Boas Práticas

  • Use apenas uma tag <main> por página.
  • Não coloque tags semânticas sem necessidade.
  • Prefira <section> e <article> com títulos claros.
